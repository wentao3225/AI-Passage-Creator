# 代码提交指南

## Commit Message 格式

- **语言**：必须使用中文编写 commit message
- **格式**：`<类型>: <主题>`

### 类型定义
- `feat`     - 新增功能
- `fix`      - 修复问题
- `docs`     - 文档更新
- `style`    - 代码风格（不影响功能）
- `refactor` - 代码重构
- `perf`     - 性能优化
- `test`     - 测试相关
- `chore`    - 构建、依赖等
- `ci`       - CI/CD 配置

### 示例
- `feat: 添加用户注册功能`
- `fix: 修复登录页面的样式问题`
- `docs: 更新 README 文档`
- `refactor: 重构用户认证模块`

## 生成规则
当使用 AI 生成 commit message 时，必须遵循以上规范，确保：
1. 使用中文描述
2. 准确反映本次提交的改动
3. 遵循 conventional commits 格式
