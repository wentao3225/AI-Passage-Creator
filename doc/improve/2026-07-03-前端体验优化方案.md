# 前端体验优化方案

> 问题日期：2026-07-03 | 涉及：前端 ArticleCreatePage.vue + 后端 ContentGeneratorAgent + SseMessageTypeEnum

---

## 问题 1：确认按钮可重复点击

### 根因

`handleConfirmOutline()` 中，`confirmLoading.value = true` 设置后立即调了 `await confirmOutline(...)`，但 `confirmOutline` API 调用完成后，**前端并没有把 `isCreating` 设为 true**。所以用户在大纲确认后、正文生成完成前，可以再次点击确认按钮。

看日志也印证了这一点：

```
09:32:27.645 大纲已确认 → executePhase3 启动
09:32:27.802 businessException: 当前阶段不允许此操作
```

用户在 Phase3 运行中再次点了确认，但后端已经处于 `CONTENT_GENERATING` 阶段，不允许再从 `OUTLINE_EDITING` 路径确认大纲。

### 修复方案

在 `handleConfirmOutline()` 中，调用 API **之前**就把按钮和操作全部锁住：

```typescript
const handleConfirmOutline = async (outlineData) => {
    confirmLoading.value = true
    isCreating.value = true          // ★ 新增：立即锁住所有操作
    try {
        await confirmOutline({ taskId: taskId.value, outline: outlineData })
        outlineRaw.value = JSON.stringify({ sections: outlineData })
        // 不再需要等 SSE 消息来解锁，因为 isCreating 已设为 true
        // 后续 SSE 消息 ALL_COMPLETE 或 ERROR 时会重置 isCreating
    } catch (error) {
        confirmLoading.value = false
        isCreating.value = false     // ★ 失败时解锁
        message.error(error.message || '确认大纲失败')
    }
}
```

### 涉及修改

| 文件 | 改动 | 风险 |
|------|------|------|
| `ArticleCreatePage.vue` | `handleConfirmOutline()` 中加 `isCreating.value = true` | 🟢 低 |

## 问题 2：工具调用时前端无反馈

### 根因

ContentGeneratorAgent 的 ReAct 循环中，Thought 和 Action 阶段是**同步阻塞调用**，没有推送任何 SSE 消息。只有 `callLlmWithStreaming()` 阶段才推送流式内容。

当前后端只发了这几种消息给前端：

| 阶段 | 后端推送 | 前端表现 |
|------|---------|---------|
| Section i, Round N Thought | ❌ 无消息 | 不动 |
| CALL_TOOL 调用工具 | ❌ 无消息 | 不动 |
| Observation | ❌ 无消息 | 不动 |
| GENERATE（流式输出） | ✅ AGENT3_STREAMING | 正常显示 |
| SGEMENT_COMPLETE | ✅ `<!-- SECTION_COMPLETE -->` | 不可见 |

### 修复方案

分两步：**后端新增 SSE 消息类型** → **前端新增消息处理**。

#### 第 1 步：后端的 ContentGeneratorAgent 增加 ReAct 进度推送

在 `processSectionWithReAct()` 的各个关键节点推送 SSE 消息。

修改 `ContentGeneratorAgent.java`，在 `processSectionWithReAct()` 中增加 `StreamHandlerContext.send()`：

```java
// 在章节开始时
StreamHandlerContext.send("REACT_SECTION_START:" + sectionTitle);

// 在 Thought 阶段（调 LLM 前后）
StreamHandlerContext.send("REACT_THOUGHT:sect=" + sectionTitle + ",round=" + (round + 1));

// 在 Action 阶段（调用工具前后）
if (thought.contains("DECISION: CALL_TOOL") && round < MAX_REACT_ROUNDS - 1) {
    String toolName = extractToolName(thought);
    String toolArgs = extractToolArgs(thought);
    String query = extractQueryFromArgs(toolArgs);
    
    // 推送：正在调用工具
    StreamHandlerContext.send("REACT_TOOL_CALL:tool=" + toolName + ",query=" + query);
    
    ToolCallResult result = toolRegistry.callTool(toolName, toolArgs);
    // ...
    
    // 推送：工具调用完成
    StreamHandlerContext.send("REACT_TOOL_RESULT:tool=" + toolName + ",success=" + result.isSuccess());
}
```

需要一个辅助方法提取搜索关键词：

```java
private String extractQueryFromArgs(String args) {
    try {
        @SuppressWarnings("unchecked")
        Map<String, String> argsMap = GsonUtils.getInstance().fromJson(args, Map.class);
        return argsMap.getOrDefault("query", "");
    } catch (Exception e) {
        return "";
    }
}
```

**新增 import**：`import com.ywt.passage.utils.GsonUtils;`（如果已有则忽略）

#### 第 2 步：后端的 ImageAnalyzerAgent 增加"正在分析"提示

`ImageAnalyzerAgent` 是同步调 LLM，中间没有进度可推，但可以在开始和结束时推消息。

当前 `ImageAnalyzerAgent` 结束时有 `StreamHandlerContext.send(AGENT4_COMPLETE + ":" + count)`，但开始时没有。在 `apply()` 开头加一行：

```java
// 在 log.info 之后、调 LLM 之前
StreamHandlerContext.send("AGENT4_ANALYZING:");
```

#### 第 3 步：前端处理新增消息类型

在 `ArticleCreatePage.vue` 的 `handleSSEMessage()` 中新增：

```typescript
// 新增：ReAct 工具调用相关状态
const currentReactSection = ref('')
const currentReactRound = ref(0)
const isSearching = ref(false)
const currentSearchQuery = ref('')

// 在 handleSSEMessage switch 中新增 case：

case 'REACT_SECTION_START':
    currentReactSection.value = msg.content || ''
    currentReactRound.value = 0
    isSearching.value = false
    addLog(`📝 开始撰写章节：${msg.content}`, 'info')
    break

case 'REACT_THOUGHT':
    currentReactRound.value = msg.round || 0
    addLog(`🤔 第 ${msg.round || 0} 轮：评估是否需要搜索资料`, 'info')
    break

case 'REACT_TOOL_CALL':
    isSearching.value = true
    currentSearchQuery.value = msg.query || ''
    addLog(`🔍 搜索资料: ${msg.query}`, 'info')
    break

case 'REACT_TOOL_RESULT':
    isSearching.value = false
    addLog(msg.success ? '✅ 搜索完成' : '⚠️ 搜索未返回有效结果', msg.success ? 'success' : 'warning')
    break

case 'AGENT4_ANALYZING':
    addLog('智能体4：正在分析配图需求...', 'info')
    break
```

#### 第 4 步：前端 CONTENT_GENERATING 阶段展示搜索状态

在 `<template>` 的 CONTENT_GENERATING 区域内，新增一个搜索状态指示器：

```html
<!-- 在正文预览之前，插入搜索状态指示 -->
<div v-if="isSearching" class="search-indicator">

        正在搜索: <code>{{ currentSearchQuery }}</code>

```

### 涉及修改

| 文件 | 改动 | 风险 |
|------|------|------|
| `ContentGeneratorAgent.java` | 在 ReAct 关键节点加 `StreamHandlerContext.send()` + `extractQueryFromArgs()` | 🟡 中（需注意流式上下文传递） |
| `ImageAnalyzerAgent.java` | 在 `apply()` 开头加一行 `StreamHandlerContext.send("AGENT4_ANALYZING:")` | 🟢 低 |
| `ArticleCreatePage.vue` | 新增 SSE 消息处理 + 模板中搜索状态指示器 + 样式 | 🟢 低 |

### 消息格式规范

所有新增消息统一格式：

| 消息 type | 格式 | 说明 |
|-----------|------|------|
| `REACT_SECTION_START` | `REACT_SECTION_START:章节标题` | 开始处理某个章节 |
| `REACT_THOUGHT` | `REACT_THOUGHT:sect=xxx,round=N` | 正在进行推理 |
| `REACT_TOOL_CALL` | `REACT_TOOL_CALL:tool=xxx,query=xxx` | 正在调工具 |
| `REACT_TOOL_RESULT` | `REACT_TOOL_RESULT:tool=xxx,success=true/false` | 工具调用完成 |
| `AGENT4_ANALYZING` | `AGENT4_ANALYZING:` | 正在分析配图 |

## 问题 3：配图分析阶段页面不动

### 根因

`AGENT3_COMPLETE` 和 `AGENT4_COMPLETE` 之间有一个**空白期**（约 30-60 秒），期间 `ImageAnalyzerAgent` 在同步调 LLM。前端收到了 `AGENT3_COMPLETE` 但下一消息是 `AGENT4_COMPLETE`，中间没有任何进度反馈。

从日志看，中间隔了大约 37 秒：

```
09:32:53.600 AGENT3_COMPLETE  → ContentGeneratorAgent 完成
09:33:28.278 ImageAnalyzerAgent 开始执行  ← 这个点没有消息推给前端
09:34:03.158 ImageAnalyzerAgent 完成  → AGENT4_COMPLETE 消息
```

### 修复方案

方案由问题 2 的第 3 步和第 4 步覆盖了——只要后端在 `ImageAnalyzerAgent` 开始时推 `AGENT4_ANALYZING`，前端收到后就能展示"正在分析配图"的状态。

前端模板已经有一个区域会在 `currentStep === 3` 时显示消息提示：

```html
<!-- 正文完成，进入配图分析步骤 -->
isStreaming.value = false
currentStep.value = 3
message.info('正文生成完成，正在分析配图...')   // 已有
addLog('智能体4：开始分析配图需求', 'info')      // 已有
```

问题 2 新增的 `AGENT4_ANALYZING` 消息只是让这个提示更准确——后端开始分析时才提示，而不是正文一结束就提示。

## 修改总结

| 问题 | 修改量 | 涉及文件 | 核心改动 |
|------|--------|---------|---------|
| 1. 按钮重复点击 | 1 行 | `ArticleCreatePage.vue` | 在 `handleConfirmOutline()` 中加 `isCreating.value = true` |
| 2. 工具调用无反馈 | ~30 行后端 + ~50 行前端 | `ContentGeneratorAgent.java` + `ArticleCreatePage.vue` | 后端在 ReAct 节点推 SSE 消息；前端新增消息处理和搜索状态指示器 |
| 3. 配图分析无反馈 | 1 行后端 | `ImageAnalyzerAgent.java` | 在 `apply()` 开头加 `StreamHandlerContext.send("AGENT4_ANALYZING:")` |

总改动量约 **80 行代码**，不涉及架构变更。
